# IAM

IAM的设计核心是**框架**和**原则**，它们在行业内有一套公认的最佳实践，用于指导企业架构的搭建。

### 1. 核心五大组件（IAM框架）

现代IAM体系由以下核心组件构成：

- **身份治理与管理 (IGA)：** 负责身份的整个生命周期管理（创建、变更、删除）。
  
- **身份验证 (Authentication)：** 确认“你是谁”。如密码、MFA（多因素认证）、生物识别等。
  
- **授权 (Authorization)：** 确定“你能做什么”。核心是访问控制模型。
  
- **目录服务 (Directory Services)：** 集中存储用户身份信息和属性（如LDAP、Active Directory）。
  
- **审计与日志 (Auditing & Logging)：** 记录所有访问活动，用于合规性检查和威胁检测。
  

### 2. 主流设计架构原则

| **原则** | **核心内容（确定内容）** |
| --- | --- |
| **最小权限原则 (PoLP)** | 授予用户完成工作所需的**最低**访问权限，这是防御内部威胁和权限提升的基石。 |
| **单点登录 (SSO)** | 允许用户使用一套凭证访问多个应用。提高了可用性，减少了密码疲劳。 |
| **多因素认证 (MFA/2FA)** | 要求用户提供两种或多种验证因素（所知、所有、所是）。这是防御凭证窃取的**最有效措施**。 |
| **零信任架构 (Zero Trust)** | **“永不信任，始终验证。”** 不再基于网络位置进行信任判断，所有用户和设备在访问资源前都必须进行身份验证和授权。 |
| **基于角色的访问控制 (RBAC)** | 基于用户的角色（如“会计”、“开发人员”）而非个人来分配权限，简化了大规模权限管理。 |
