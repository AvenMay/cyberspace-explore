# Unpackers

#### **A. 动态调试与转储 (Debuggers & Dumpers)**

这是手动脱壳的核心工具链：**寻找 OEP (原始入口点) -> Dump 内存 -> 修复 IAT (导入地址表)**。

- **x64dbg (x32dbg/x64dbg):** 目前 Windows 平台最主流的调试器。
  
    - *用途:* 动态调试，绕过反调试检测，手动寻找 OEP。
      
    - *插件:* **ScyllaHide** (隐藏调试器)，**Scylla** (内置的强大的 Dump 和 IAT 修复工具)。
    
- **Scylla:** 独立的 IAT 修复工具（也被集成在 x64dbg 中）。它是脱壳流程中修复导入表的标准工具。
  
- **OllyDbg:** 经典的 32 位调试器，虽然仍在某些老旧样本分析中使用，但逐渐被 x64dbg 取代。
  

#### **B. 专用/自动化脱壳机 (Automated Unpackers)**

针对特定壳的工具，能一键脱壳，但通用性差。

- **UPX (`-d` 选项):** UPX 自带解压功能 (`upx -d file.exe`)。
  
- **UnpacMe:** 一个基于 Web 的自动化脱壳服务（商业/社区），上传样本后它会自动尝试识别并脱壳，非常强大。
  
- **QuickUnpack:** 尝试通用脱壳的老牌工具（对现代强壳效果有限）。
  
- **De4dot:** 专门针对 **.NET** 程序的脱壳和反混淆神器（支持多种 .NET 混淆器）。
  

#### **C. 高级模拟与分析框架**

- **Qiling Framework:** 这种是基于 Python 的高级二进制模拟框架，常用于模拟执行壳的代码片段来辅助脱壳。
  
- **Unicorn Engine:** CPU 模拟引擎，许多高级脱壳脚本底层都依赖它。
