# Application Security Testing

应用安全测试的方法有多种，通常根据测试的阶段、目标和技术的不同，可以分为几种主要的测试方法。以下是常见的应用安全测试方法：

> **SAST, DAST, IAST, Fuzzing** 都是**应用安全测试**的核心方法，专注于应用层的安全性。
> 
> **Pentesting** 和 **SCA**，尽管涉及应用安全测试，但它们通常跨越多个安全层次，包括网络、操作系统和其他基础设施的安全性。

### 1. **静态应用安全测试（SAST，Static Application Security Testing）**

- **方法概述**：静态应用安全测试（SAST）是一种在应用源代码、字节码或二进制文件中进行分析的方法。它在程序运行之前对代码进行扫描，目的是发现代码中潜在的安全漏洞，如SQL注入、XSS（跨站脚本）等。
- **特点**：
  - **早期发现漏洞**：SAST方法通常应用于开发阶段，能够在软件开发生命周期早期发现安全问题。
  - **代码级分析**：直接分析源代码或字节码，不需要运行程序。
  - **自动化程度高**：常用自动化工具扫描源代码和静态文件，能提高效率。
- **常用工具**：SonarQube、Checkmarx、Fortify、Veracode。

### 2. **动态应用安全测试（DAST，Dynamic Application Security Testing）**

- **方法概述**：动态应用安全测试（DAST）是一种黑盒测试方法，测试人员将应用程序部署到运行环境中，通过模拟攻击进行安全测试。DAST主要是对运行中的应用程序进行扫描，找出安全漏洞。
- **特点**：
  - **运行时漏洞检测**：与SAST不同，DAST关注的是运行时的安全问题，比如输入验证、认证、会话管理等。
  - **无需源代码**：DAST不需要访问源代码，只需要运行中的应用或服务。
  - **模拟攻击**：DAST通过模拟外部攻击者的行为来发现漏洞。
- **常用工具**：OWASP ZAP、Burp Suite、Acunetix、Nessus。

### 3. **交互式应用安全测试（IAST，Interactive Application Security Testing）**

- **方法概述**：交互式应用安全测试（IAST）结合了SAST和DAST的特点，它通过在应用运行时进行实时分析，检测并识别漏洞。IAST通过在应用服务器中嵌入一个代理，实时监控应用的运行情况，并分析源代码和数据流。
- **特点**：
  - **实时检测**：IAST在应用运行过程中对代码和数据流进行实时监控，发现安全问题。
  - **开发和测试阶段结合**：适用于开发和测试阶段，结合了静态分析和动态分析的优势。
  - **精准度高**：相比于SAST和DAST，IAST能提供更精准的漏洞检测，因为它不仅分析代码，还分析运行时的行为。
- **常用工具**：Contrast Security、Seeker、Veracode IAST。

### 4. **软件组合分析（SCA，Software Composition Analysis）**

- **方法概述**：软件组合分析（SCA）关注的是应用中使用的第三方库、框架和组件的安全性。通过扫描代码中引用的开源库或第三方组件，检查这些组件是否存在已知的安全漏洞。
- **特点**：
  - **开源和第三方组件检测**：SCA主要用于检测应用使用的开源软件组件及其依赖库中的安全漏洞。
  - **库和组件管理**：随着应用依赖大量外部组件，SCA能够有效识别外部依赖中的漏洞和不安全的库。
  - **合规性检查**：SCA还可以检测应用中使用的开源组件是否符合合规要求。
- **常用工具**：OWASP Dependency-Check、Snyk、Black Duck、Sonatype Nexus.

### 5. **模糊测试（Fuzz Testing）**

- **方法概述**：模糊测试（Fuzz Testing）是一种通过向应用程序输入随机或异常的数据来检测程序异常行为或崩溃的测试方法。模糊测试能够有效地发现应用中的内存泄漏、缓冲区溢出、数据处理错误等漏洞。
- **特点**：
  - **异常输入测试**：通过输入异常数据（如随机数据、极端数据等）来暴露应用程序的漏洞。
  - **自动化工具**：模糊测试常常由自动化工具执行，可以快速检测出潜在的安全漏洞。
  - **高效发现漏洞**：能够暴露一些复杂的安全漏洞，尤其是与内存相关的漏洞。
- **常用工具**：AFL（American Fuzzy Lop）、Peach Fuzzer、Burp Suite Intruder。

### 6. **手动安全测试**（Penetration-Testing）

- **方法概述**：手动安全测试是一种由安全专家进行的渗透测试方法。测试人员模拟黑客攻击，手动验证漏洞，深入分析应用程序的安全性。这种方法通常用于发现SAST和DAST工具可能漏掉的复杂漏洞。
- **特点**：
  - **灵活性高**：测试人员可以根据情况进行灵活的攻击模拟，能够发现工具无法识别的漏洞。
  - **技术性强**：需要安全专家的深厚技术背景和经验。
  - **模拟实际攻击**：测试人员通过手动攻击模拟真实的黑客行为，评估应用程序的防御能力。
- **常用工具**：Burp Suite、Metasploit、Wireshark。

### 7. **黑盒测试、白盒测试与灰盒测试**

这三种测试方法分别对应不同的测试角度：

- **黑盒测试**（Black-box Testing）：测试者不考虑应用的内部结构，只关注其外部行为，常用的有DAST和手动渗透测试。
- **白盒测试**（White-box Testing）：测试者拥有应用的源代码或架构设计，进行详细的代码级别的审查，常用的有SAST和IAST。
- **灰盒测试**（Gray-box Testing）：测试者部分了解应用的内部结构和代码实现，但不是全部，常结合黑盒和白盒方法使用。

### 总结

应用安全测试的方法根据测试的侧重点和所用的工具不同，常见的有：

- **SAST**：静态代码分析，提前发现代码层面的漏洞。
- **DAST**：动态分析，运行时测试，模拟外部攻击。
- **IAST**：结合SAST和DAST，实时监控应用运行，发现潜在漏洞。
- **SCA**：检查开源组件和第三方库中的安全漏洞。
- **模糊测试**：输入异常数据，测试系统的异常行为和漏洞。
- **手动渗透测试**：模拟实际黑客攻击，测试复杂的、自动化工具难以发现的漏洞。
- **黑盒、白盒和灰盒测试**：从不同角度进行应用安全测试。

这些方法可以根据应用的开发阶段、测试目标和具体需求进行选择和组合。
