# Internet System

把整个Internet-System的技术架构，简化为Application层面、Infrastructure层面，以及IoT层面，是从功能和角色划分的角度来理解复杂系统。


## Application
这是直接面向用户或业务的部分，包括各种软件应用、服务、网站、移动App等。负责将基础资源抽象成业务功能，满足用户需求。这层关注的是用户体验、业务逻辑、功能实现。它依赖于Infrastructure层提供的资源和能力，但更注重业务价值。

- **核心使命**：满足业务需求，实现具体功能。
- **关注点**：功能模块化、组件协作、业务流程设计、用户体验。
- **特点**：多样化、面向领域，依赖基础设施提供稳定可靠的环境。
- **例子**：电商平台的订单系统、支付系统，社交软件的消息模块，企业的CRM系统。


## Infrastructure
这是所有计算和网络活动的底层支撑，包括硬件（服务器、存储、网络设备、数据中心）、操作系统、虚拟化技术、云平台等。负责提供计算、存储、网络等基础资源，是整个生态的“地基”。没有这一层，应用无法运行，数据无法传输。这层关注的是可靠性、扩展性、性能、安全。

- **核心使命**：将底层复杂且多样的硬件和资源抽象成简洁、统一的服务能力。
- **三个核心主题**：
    - 计算：CPU、GPU、虚拟机、容器、边缘计算等
    - 存储：数据库、对象存储、分布式文件系统等
    - 网络：物理网络、虚拟网络、SDN、传输协议等
- **特点**：隐蔽复杂性，强调高性能、弹性、可用性、安全性。
- **例子**：操作系统、云计算平台（AWS、Azure、阿里云）、数据中心、网络交换机。


## Embedded/IoT

IoT层是在Internet和计算技术应用中越来越重要的一环。主要是指连接各种传感器、设备、智能硬件，收集现实世界的数据。
这层既是数据的产生者，也是应用和基础设施的延伸，常常需要独特的通信协议、实时处理、边缘计算支持。在很多传统IT系统中不一定显式，但在现代智能化、工业互联网、智能家居等场景必不可少。
核心使命：将底层复杂且多样的硬件和资源抽象成简洁、统一的服务能力。

- **核心使命**：把现实世界的设备、传感器、终端与数字系统有效连接。
- **特点**：领域广泛，技术多样（无线通信、边缘计算、协议多样性）。
- **作用**：作为Application与Infrastructure之间或之外的“桥梁”，扩展了整个系统的边界。
- **例子**：智能家居设备、工业传感器、智能交通系统。

## 为什么用这三个层面来简化架构？
分层有助于复杂性管理：计算机系统极其复杂，分层帮助人们专注不同职责，分别优化和发展。清晰的职责分离：每层关注的核心问题不同，减少耦合，提高模块化和灵活性。适应技术演进：新技术（如IoT）可以作为独立层面插入，不破坏已有架构。方便设计和沟通：设计者、工程师、业务方可以围绕这三个层面明确讨论需求和实现。

## 拓展理解
其实，这样的分层思想类似于经典的网络分层模型（OSI七层），以及现代云计算的架构设计思路。比如云原生架构就把基础>设施（IaaS/PaaS）和应用（SaaS）层面分开，IoT则是介于物理世界和数字世界的桥梁。
